### [约 01:15]
**问题：** r183 底部动画面板应该只在选中有动画的对象时显示，未选中或选中无动画对象时应隐藏
**解决：** r183 的 Animation.js 初始化时无条件显示面板，且 objectSelected 回调只处理有动画的情况不处理隐藏。在两个 bootstrap 的 tryHideAnimation 中增加 objectSelected 监听，当选中对象无动画时强制隐藏面板和 resizer 并 dispatch false。
**涉及文件：** `plugin/bootstrap/verse-bootstrap.js`, `plugin/bootstrap/meta-bootstrap.js`

### [约 01:30]
**问题：** 未选中有动画对象时，底部动画面板的白色 resizer 条仍然显示
**解决：** r183 的 Animation.js 的 `update()` 会被 `objectAdded`/`objectRemoved` 信号触发，无条件 dispatch panelHeight 导致 resizer 重新显示。在两个 bootstrap 中增加 `animationPanelChanged` 信号拦截器，当 height 不为 false 时检查当前选中对象是否有动画，无动画则强制隐藏面板和 resizer。
**涉及文件：** `plugin/bootstrap/verse-bootstrap.js`, `plugin/bootstrap/meta-bootstrap.js`
---

---
### [约 01:45]
**问题：** 通过 inline style 设置 `display: none` 无法可靠隐藏动画面板的 resizer 白色条
**解决：** r183 的 AnimationResizer 通过信号监听器不断用 inline style 覆盖 `display` 值，导致 JS 层面的 `style.display = 'none'` 被后续信号处理覆盖。改用 CSS class `.mrpp-animation-hidden { display: none !important; }` 配合 `classList.toggle()` 来控制显隐，`!important` 确保优先级高于任何 inline style。
**涉及文件：** `plugin/bootstrap/verse-bootstrap.js`, `plugin/bootstrap/meta-bootstrap.js`
---

---
### [约 02:00]
**问题：** CSS class + !important 方案仍未能隐藏 animation-resizer 白色条
**解决：** 根本原因是 `#animation` 和 `#animation-resizer` 在 HTML 中按顺序添加到 DOM，MutationObserver 在 `#animation` 添加时触发 tryHideAnimation 并设置 `animationHidden = true`，此时 `querySelectorAll('#animation-resizer')` 返回空 NodeList（resizer 还没创建）。后续 resizer 添加时因 `animationHidden` 已为 true 直接跳过。修复为 `setAnimationHiddenClass` 每次调用时用 `getElementById` 重新查询而非依赖缓存的 NodeList。
**涉及文件：** `plugin/bootstrap/verse-bootstrap.js`, `plugin/bootstrap/meta-bootstrap.js`
